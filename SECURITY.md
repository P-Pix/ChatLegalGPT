# Security Policy

## Signalement de vulnérabilités
Merci de ne pas ouvrir d’issue publique pour une vulnérabilité.  
Préférez un contact privé (email / message) avec :
- description
- impact
- étapes de reproduction

## Bonnes pratiques
- Ne pas mettre de clés API (PISTE, etc.) dans le dépôt
- Utiliser des variables d’environnement / `.env` (non versionné)
